# 应用验证指南

## 一、验证方式选择

| 方式 | 命令 | 适用场景 |
|------|------|----------|
| **DMG 安装包** | 双击 `.dmg` → 拖入 Applications | 最终验收（5.6） |
| **Tauri 开发模式** | `cd frontend && pnpm tauri dev` | 含 sidecar 完整链路 |
| **纯前端开发模式** | 后端 `uvicorn` + 前端 `pnpm dev` | 日常开发调试 |

---

## 二、Sidecar 独立验证（G3 补充）

在终端执行：

```bash
# 启动 sidecar（用非占用端口）
frontend/src-tauri/binaries/backend-aarch64-apple-darwin \
  --host 127.0.0.1 --port 17398 --token test123 &
sleep 3

# 验证 health API
curl -sf -H "Authorization: Bearer test123" http://127.0.0.1:17398/api/health
# 预期输出：{"status":"ok"}

# 清理
kill %1
```

## 三、DMG 安装包验证

### 安装

```bash
# DMG 路径
open frontend/src-tauri/target/release/bundle/dmg/price-comparison-scanner_0.1.0_aarch64.dmg
```

双击 DMG → 将 `price-comparison-scanner.app` 拖入 Applications → 从启动台打开。

> macOS 首次打开可能提示"无法验证开发者"，右键 → 打开 → 确认即可。

### 验证要点

- 应用窗口正常显示
- 底部或日志中无"sidecar 启动失败"提示
- 可以正常新建项目

## 四、开发模式验证（推荐日常使用）

### 启动

```bash
# 终端 1：后端
cd backend
uvicorn main:app --host 127.0.0.1 --port 17396 --reload

# 终端 2：前端
cd frontend
pnpm dev
```

浏览器打开 http://localhost:5173

## 五、核心流程验证（5 阶段工作台）

### 测试数据位置

```
tests/e2e/test-data/
├── ds-1/   3 个 xlsx（IT 办公设备，标准场景）
├── ds-2/   xlsx + docx + pdf（办公家具，混合格式）
├── ds-3/   4 个 xlsx（实验室耗材，异常场景）
├── ds-4/   3 个 xlsx（网络设备，归组挑战）
└── ds-5/   xlsx + 扫描版 pdf（OCR 路径）
```

**建议先用 DS-1 跑通完整流程，再用 DS-3 验证异常检出。**

---

### 阶段 1：导入文件

1. 点击「新建项目」，输入项目名称
2. 点击「导入文件」或拖拽文件到导入区
3. 使用 `tests/e2e/test-data/ds-1/` 下的 3 个 xlsx 文件
4. 等待解析完成（每个文件 < 1 秒）

**验证**：
- 3 个文件全部显示「解析完成」
- 供应商名称自动猜测（联想科技 / 华硕经销 / 戴尔直销）
- 点击每个文件的「确认供应商」按钮

### 阶段 2：标准化

1. 切换到「标准化」Tab
2. 点击「执行标准化」
3. 等待完成（异步任务，< 1 秒）

**验证**：
- 表格显示标准化后的数据
- 列名已映射为标准字段（商品名称、规格型号、单价等）
- 如有未识别列名，可手动修正

### 阶段 3：商品归组

1. 切换到「商品归组」Tab
2. 点击「开始智能归组」
3. 等待完成（< 1 秒）

**验证**：
- 左侧显示候选归组列表（按置信度分组）
- DS-1 应有 5 个归组（笔记本电脑、显示器、键鼠、扩展坞、打印机）
- 点击「一键确认全部」确认所有归组

### 阶段 4：符合性审查（可选）

- 此阶段可跳过，直接进入比价导出
- 如需测试：录入需求标准 → 执行符合性审查

### 阶段 5：比价导出

1. 切换到「比价导出」Tab
2. 点击「生成比价」
3. 等待比价计算完成
4. 查看比价结果表格
5. 点击「导出底稿」

**验证**：
- 比价结果表格显示各供应商价格对比
- 最低价高亮标记
- 导出成功，提示 Excel 文件路径
- 导出的 Excel 包含 3 个 Sheet（比价汇总、明细数据、追溯信息）

## 六、异常场景验证（DS-3）

使用 `tests/e2e/test-data/ds-3/` 的 4 个文件，验证系统能检出：

| 异常 | 说明 |
|------|------|
| 税价口径不一致 | A 含税、B 不含税、C 未标注 |
| 单位不一致 | 移液器组：支 vs 把 |
| 币种不一致 | Sigma 用 USD，其他用 RMB |
| 必填字段缺失 | 供应商 C 某行单价为空 |

**验证**：比价结果中应有异常标记，问题面板显示异常汇总。

---

## 七、归组质量验证（DS-4）

使用 `tests/e2e/test-data/ds-4/` 的 3 个文件，验证品牌别名替换：

| 原始 | 归一化后 |
|------|----------|
| Cisco | 思科 |
| HUAWEI / Huawei | 华为 |
| HP | 惠普 |
| Hikvision | 海康威视 |

**验证**：归组结果应显示 6 个组（交换机、无线AP、服务器、路由器、摄像头、NVR），归组名称含中文品牌名。

## 八、OCR 降级验证（DS-5）

使用 `tests/e2e/test-data/ds-5/` 的 2 个文件：

- `标准供应商A.xlsx` — 应正常解析
- `扫描版报价单.pdf` — 应触发 OCR 降级提示

**验证**：应用不崩溃，提示 OCR 扩展未安装，xlsx 文件不受影响。

## 九、问题排查

| 症状 | 可能原因 | 解决 |
|------|----------|------|
| 应用打开白屏 | sidecar 未启动 | 检查后端进程是否存在 |
| API 调用 401 | token 不匹配 | 重启应用 |
| 标准化/归组无响应 | 异步任务卡住 | 检查后端日志 |
| 导出失败 | 归组未确认 | 回到归组阶段点击确认 |
| DMG 打开被阻止 | macOS Gatekeeper | 右键 → 打开 → 确认 |
| sidecar 安全模式 | 后端多次崩溃 | 关闭应用重新启动 |
