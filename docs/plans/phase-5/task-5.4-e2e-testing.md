### 验收执行流程

对每套数据集，按以下步骤执行：

```
1. 新建项目（命名为 DS-X 测试）
2. 导入所有测试文件
3. 确认供应商名称
4. 选择参与比价的表格
5. 执行标准化
6. 检查列名映射结果 → 记录自动映射命中率
7. 人工修正需要修正的项
8. 执行归组
9. 检查归组结果 → 记录置信度分层是否正确
10. 确认归组（人工确认中置信组）
11. （DS-4 含需求标准的场景）录入需求标准 → 执行符合性审查
12. 执行比价
13. 检查异常标记 → 记录异常检出率
14. 导出 Excel
15. 检查导出内容：Sheet 数量、追溯信息、比价结果
16. 记录验证结果
```

### 各数据集验收标准

#### DS-1 验收清单

- [ ] 3 个 xlsx 文件均成功解析
- [ ] 供应商名称从文件名正确猜测
- [ ] 列名自动映射命中率 >= 80%（预期 100%）
- [ ] 所有归组为中置信（品牌不同触发硬约束）
- [ ] 用户确认归组后比价结果正确（5 组最低价与预期一致）
- [ ] 导出 Excel 包含 3 个 Sheet（比价汇总、明细数据、追溯信息）
- [ ] 追溯信息表每行可追溯到来源 sheet + 单元格

#### DS-2 验收清单

- [ ] xlsx、docx、数字版 PDF 三种格式均成功解析
- [ ] Word 文档中表格前的说明文字被正确忽略
- [ ] PDF 结构化提取成功（L1）
- [ ] 「参数」「计量」「货期」等非典型列名标记为待确认
- [ ] 供应商 C 的「不含税单价」被识别为 `tax_basis = known_exclusive`
- [ ] 税价口径不一致异常被正确检出（5/5 = 100%）
- [ ] 高置信归组（组 1/2/3/5）无需用户修改

#### DS-3 验收清单

- [ ] 4 个供应商文件均成功导入
- [ ] 8 个预设异常全部被系统检出（E1-E8）：
  - E1: 税价口径不一致（A 含税、B 不含税、C 未标注、D 0%）
  - E2: 税价口径未知（供应商 C 所有行）
  - E3: 单位不一致（移液器组：支/把）
  - E4: 单位不一致（离心管组：袋/包/箱）
  - E5: 币种不一致（D 为 USD）
  - E6: 必填字段缺失（供应商 C 移液枪单价为空）
  - E7: 缺项（D 仅报 2 个品种）
  - E8: 交期字段异常（供应商 C 交期列为空）
- [ ] 税价口径不一致时不给出最低价结论
- [ ] 单位不一致时不做价格比较
- [ ] 待处理问题清单显示所有异常汇总

#### DS-4 验收清单

- [ ] 品牌别名替换全部正确（cisco/huawei/hp/hikvision）
- [ ] 噪音词移除不误伤核心名称
- [ ] 高置信归组（组 1/2/3/5/6）全部自动候选，无需用户修改
- [ ] 中置信归组（组 4：路由器）正确降级为待确认
- [ ] 归组理由文本可读且准确
- [ ] 高置信归组无需修改率 >= 90%

#### DS-5 验收清单

- [ ] 标准 xlsx 正常解析
- [ ] 扫描版 PDF 触发 OCR 降级路径
- [ ] 显示「OCR 扩展未安装」提示
- [ ] 应用不崩溃，其他文件不受影响
