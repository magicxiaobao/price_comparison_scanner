## 实现规格

### 第一部分：安装包验证

在目标平台上执行安装包的完整安装和运行验证。

#### macOS 验证步骤

```
1. 双击 .dmg 文件
2. 将应用拖到 Applications 目录
3. 从 Applications 启动应用
   - 首次启动可能提示「无法验证开发者」→ 右键打开 / 系统设置允许
4. 验证 sidecar 自动启动
   - 应用加载后可正常操作
   - 活动监视器中可见 backend 进程
5. 完成基础流程（见下文）
6. 正常关闭应用
   - backend 进程消失
   - PID 文件被清理
```

#### Windows 验证步骤

```
1. 双击 .msi 文件
2. 按安装向导完成安装
3. 从开始菜单 / 桌面快捷方式启动应用
   - SmartScreen 可能提示 → 选择「仍要运行」
4. 验证 sidecar 自动启动
   - 任务管理器中可见 backend 进程
5. 完成基础流程
6. 正常关闭应用
7. 通过「添加/删除程序」卸载 → 确认清理干净
```

#### Linux 验证步骤

```
1. 赋予 .AppImage 执行权限：chmod +x *.AppImage
2. 直接运行 .AppImage
3. 验证 sidecar 自动启动
4. 完成基础流程
5. 正常关闭应用
```

### 第二部分：安装包基础流程验证

使用 DS-1 数据集在安装包环境中跑通完整流程：

```
1. 新建项目「安装包验证」
2. 导入 3 个 xlsx 文件（DS-1）
3. 确认供应商名称
4. 选择表格
5. 执行标准化 → 检查结果
6. 确认归组
7. 执行比价
8. 导出 Excel → 检查输出文件
9. 关闭项目
10. 重新打开项目 → 验证数据持久化
```
