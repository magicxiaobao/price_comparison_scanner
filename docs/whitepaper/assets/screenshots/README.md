# 截图资源目录说明

本目录存放白皮书所用的全部界面截图资产。

---

## 目录用途

本目录用于集中存放"三方比价支出依据扫描工具"产品宣传白皮书中使用的真实运行界面截图和导出结果截图。所有截图须符合以下标准后方可嵌入白皮书正文。

---

## 命名规范

截图文件统一采用两位数字前缀 + 语义名称的方式命名，扩展名统一使用 `.png`：

```
<序号>-<语义名称>.png
```

示例：

| 文件名 | 说明 |
|--------|------|
| `01-home.png` | 首页 / 最近项目列表 |
| `02-create-project-dialog.png` | 新建项目弹窗 |
| `03-import-stage.png` | 文件导入阶段 |
| `04-standardize-stage.png` | 标准化阶段表格 |
| `05-grouping-stage.png` | 商品归组界面 |
| `06-compliance-stage.png` | 符合性审查界面 |
| `07-comparison-stage.png` | 比价结果与导出 |
| `08-rule-management.png` | 规则管理页面 |
| `09-app-preferences.png` | 应用设置页面 |
| `10-export-result.png` | 导出后的底稿或导出提示 |

---

## 图片来源优先级

截图来源按以下优先级使用：

1. **第一优先：真实运行界面截图**
   - 通过 Playwright 访问本地 Web 应用（开发模式），截取纯内容区域
   - 视口固定为 `1600x1000`，`deviceScaleFactor=2`
   - 使用统一样例数据，保持项目状态一致
   - 不包含浏览器地址栏、系统桌面外框或 Tauri 窗口边框

2. **第二优先：真实导出结果截图**
   - 对 Excel 审计底稿等导出文件进行截图
   - 同样须裁剪至内容区域，去除系统桌面背景

3. **第三优先：设计图补位**
   - 来源：`docs/design/ui/*/screen.png`
   - 仅在真实界面无法稳定复现时使用
   - 使用设计图时，必须在本 README 的截图清单中标注来源为 `design-fallback`
